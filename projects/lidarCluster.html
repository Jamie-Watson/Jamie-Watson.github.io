Data Clustering using LiDAR

June 4th, 2024

School Project Java Based

Project Goals

This project was a programming assignment for my Data Structures and Algorithms course (CSI2110), which consisted of experimenting with the DBSCAN algorithm to cluster points. The Java side of the program is composed of four classes: DBScan, NearestNeighbor, Point3D, and a Stack implementation, to utilize the DBSCAN algorithm.

The primary objective of this assignment was to efficiently and accurately process points in a 3D space and cluster them into distinct objects. DBSCAN, or Density-Based Spatial Clustering of Applications with Noise, which works well as it can identify clusters of varying shapes and sizes.

To complement the Java implementation, the assignment also includes a Python visualizer. This visualizer enables viewing the clusters in 3D, providing a representation of the clusters which have been grouped into distinct objects.

Class Purpose
1 DBScan Handles the core clustering logic through the main and runs DBScan algorithm
2 NearestNeighbors Helper class which quickly finds points within a specified radius
3 Point3D Creates the 3D point objects which represent individual points in 3D space
4 Stack Implements the stack data structure to aid with running the DBScan algorithm

What is DBScan?

The entire project revolves around the DBSCAN algorithm, but what is its purpose? Autonomous navigation in vehicles is likely to become prominent in the future, with one implementation being explored through Lidar (Light Detection And Ranging) sensors. Lidar sensors scan the surrounding area by recording the time it takes for a laser to bounce back to its origin, generating a 3D point. With a large number of such points, a detailed scene can be recorded. The challenge then becomes how to separate these 3D points in space into distinguishable objects (e.g., a person walking, a building, a tree). The DBSCAN algorithm addresses this by processing these points and clustering them into distinct objects.

The DBSCAN algorithm takes in three parameters: DB, the dataset of all 3D points; eps, the minimum distance for points to be considered neighbors; and minPts, the minimum number of points in a region to be considered dense. The DBSCAN algorithm works by iterating through all the points in the dataset and labeling them based on their density. Points with at least minPts close neighbors are considered to be in a high-density area, while points that do not meet this criterion are labeled as noise points. After this initial labeling, the algorithm clusters all high-density points that are within the eps distance of each other, forming distinct clusters.
